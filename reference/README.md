# Reference Directory

Place genome FASTA, GTF/GFF annotation files, and STAR indexes here.

Large reference files and generated STAR indexes are NOT tracked by git.
Generate or download them before running the pipeline.

## Quick Start: chr22 Demo Reference

A minimal chr22 reference (~51 Mb) is enough to test the pipeline end-to-end:

```bash
# Download chromosome 22 FASTA and GTF (GRCh38)
mkdir -p reference/chr22
cd reference/chr22

# FASTA
wget -c ftp://ftp.ensembl.org/pub/release-113/fasta/homo_sapiens/dna/Homo_sapiens.GRCh38.dna.chromosome.22.fa.gz
gunzip Homo_sapiens.GRCh38.dna.chromosome.22.fa.gz

# GTF
wget -c ftp://ftp.ensembl.org/pub/release-113/gtf/homo_sapiens/Homo_sapiens.GRCh38.113.chr.gtf.gz
gunzip Homo_sapiens.GRCh38.113.chr.gtf.gz
# Keep only chr22 lines
grep -P '^(#|22\t)' Homo_sapiens.GRCh38.113.chr.gtf > chr22.gtf
rm Homo_sapiens.GRCh38.113.chr.gtf

# Build STAR index
mkdir -p star_index
STAR --runMode genomeGenerate \
     --runThreadN 8 \
     --genomeDir star_index \
     --genomeFastaFiles Homo_sapiens.GRCh38.dna.chromosome.22.fa \
     --sjdbGTFfile chr22.gtf \
     --sjdbOverhang 100

cd ../..
```

## Full Reference Setup

Replace paths in `config/config.yaml` when using a full genome:

```yaml
reference:
  star_index: "reference/GRCh38/star_index"
  gtf: "reference/GRCh38/gencode.v44.annotation.gtf"
  genome_fasta: "reference/GRCh38/GRCh38.p14.genome.fa"
```

## Reference Directory Structure

```
reference/
├── README.md
├── chr22/                        # Demo: chr22 only (~51 Mb)
│   ├── Homo_sapiens.GRCh38.dna.chromosome.22.fa
│   ├── chr22.gtf
│   └── star_index/               # Generated by STAR genomeGenerate
│       ├── Genome
│       ├── SA
│       ├── SAindex
│       └── ...
└── GRCh38/                       # Production: full genome
    ├── GRCh38.p14.genome.fa
    ├── gencode.v44.annotation.gtf
    └── star_index/
```
